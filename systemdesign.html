System Design

System Architecture Diagram




The system can be subdivided into five distinct components which interact with each other. These consist of:

  1. User: The user is the person who engages with the front-end interfaces and various input devices to interact with the system.
  2. Input Devices: These devices capture the user's actions and provide audiovisual data to the back-end for processing and interpretation.
  3. Motion-Input Front-End: This component allows users to customize their preferences in the MFC menu, such as choosing their preferred input mode (e.g., hands or nose), and modifying speech trigger phrases. It also displays camera visuals to the user, assisting them in controlling MotionInput.
  4. MotionInput Back-End: Responsible for processing and translating the audio and visual data from input devices into actionable system commands. The KITA module manages audio data using the VOSK library, while the ChangeDictionary class adjusts the number of words KITA can recognize. The back-end also houses the CADEditor Module, which ensures that appropriate events and gestures are loaded based on the user's chosen mode.
  5. Data Store: This component serves as the storage for all system data, enabling the back-end to access and manipulate JSON data for various read and write operations.
